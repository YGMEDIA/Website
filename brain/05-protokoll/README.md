# 05-protokoll/ — Das Gedächtnis fürs Detail

> Hier wird dokumentiert, WAS gemacht wurde und WIE — dauerhaft, datiert, kategorisiert.
> STATUS.md bleibt der kompakte aktuelle Stand (Verlauf = 1–3 Sätze pro Schritt + Verweis hierher). Die Tiefe lebt hier.

## Regeln
1. **Ein Eintrag pro abgeschlossenem Schritt** — Dateiname `JJJJ-MM-TT-<kategorie>-<thema>.md`. Kategorien: `dev` (Technik/Struktur) · `content` (Seiten/Texte/SEO) · `marketing` (Ads, Outreach, GSC-Aktionen) · `system` (Brain, Prozesse, Werkzeuge).
2. **Pflicht-Inhalt:** Was (Ergebnis) · Wie (Vorgehen, Patterns/Skripte) · Warum so (falls Alternativen verworfen) · Verify (womit bewiesen) · Gelernt (→ floss es in Constitution/Patterns zurück?).
3. **Wer schreibt:** Claude Code nach jedem Bau-Schritt automatisch (steht in CLAUDE.md). Yasin bzw. Chat-Claude für Offline-/Dashboard-Aktivitäten — Stichpunkte reichen, Hauptsache datiert.
4. Einträge werden nie umgeschrieben, nur ergänzt (Nachtrag mit Datum). Falsches wird als korrigiert markiert, nicht gelöscht — das Protokoll ist Historie, kein Wiki.
