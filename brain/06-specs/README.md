# 06-specs/ — Feature-Specs

Größere Vorhaben (neue Seiten, Struktur-Umbauten, alles mit Geld/Recht) werden nur nach freigegebenem Spec gebaut (P-9, Framework 5.3). Kleine Fixes (<5 Dateien, kein Geld/Recht/Löschen) brauchen kein Spec, aber verify.py + STATUS + Protokoll immer.
Format: `SPEC-<name>.md` mit Ziel · Keyword-/Seiten-Bezug · Änderungen DE+EN · Gesetzes-Check · Produktentscheidungen für Yasin mit [CC-Empfehlung] · Verify-Gate.
